HEAT Discovery Asset Management

The HEAT Discovery page enables you to deploy Discovery in various ways.

This is an advanced function for administrators that is available from the Configuration Console. Click Configure Application to view the Configuration Console.
Access Asset Management
1. From the Configuration Console Settings pane under Extend, click HEATDiscovery.

The HEATDiscovery installers appear.

HEAT Discovery Page

2. Click the name of an installer to begin the installation.
Windows MSI - Administrative Deployment Option

This package enables easy mass deployment of the client via software distribution tools such as DSM, SCCM or Active Directory policies. Using this method, the Discovery client is automatically installed to all clients logging into the domain. Their inventory data is then sent to HEAT.

When you are installing the gateway, you need to provide the following information:

  • The name of the gateway (this is usually populated automatically).
  • The name of the domain on which the gateway will install clients.
  • Any associated subnets.
  • The local path of the HEAT agent deployment share.
  • The account name and password on the computers to which the gateway deploys. The account must have administrative privileges on the client computers.
  • The client access key. The HEAT gateway installer package contains a unique client access key used to authenticate incoming data from the customer. (This is provided automatically during the installation.)

The permissions to the HEAT agent deployment share are set to allow domain computers and users read access.

1. Optional. Check Associate with an OU upon installation, then choose an organizational unit. The organizational unit is automatically associated upon installation.
When specifying a default organizational unit when installing the gateway, ensure that the computer that you are using to install the gateway does not already have a CI record in HEAT.
2. Click the title to download the MSIPackage file to your local machine. The download confirmation window appears.
3. Choose Save File to place it into your preferred directory, then click OK. The file downloads.
4. Extract the zip file and follow the instructions in the enclosed readme file.
Run Windows Client Agent Installer

This package enables you to install client agents on remote (not consistently connected to your intranet) computers running Microsoft Windows.

The client agent installation requires local administrator permissions.

1. Optional. Check Associate with an OU upon installation, then choose an organizational unit. The organizational unit is automatically associated upon installation.
When specifying a default organizational unit when installing the gateway, ensure that the computer that you are using to install the gateway does not already have a CI record in HEAT.
2. Click the title to download the ClientSetup file to your local machine. The download confirmation window appears.
3. Choose Save File to place it into your preferred directory, then click OK. The file downloads.
4. Provide the file to your remote users (generally via email). The users execute the file from their computers.
