Importing and Exporting Data

The HEAT architecture easily connects to and integrates with data sources and simplifies the importation of external data from other web services.

Use the Data Integration WizardUsed to import data from external sources into HEAT. to import data from external sources into HEAT. Importing records is an important activity when setting up your Service Desk. In HEAT, importing records helps to quickly create new employees, incidents, tasks or other business object records.

The Data Integration Wizard allows users to:

  • Map the source file to the target business object while setting up the import instead of setting up the mapping prior to importing as in the existing business object uploader.
  • Specify a template file to set up the mapping if the actual data file is too large.
  • View details of all the existing schedules in HEAT when setting up the import.

Integration enables you to set up and maintain data across two systems.

From the Integration workspace, you can access your configured import files. You can schedule imports to run at specific times, or manually import data as needed.

How Data Integration Works

  • You have a file containing the data to import. The file can be in .CSV, .XLS, or .XLSX (Microsoft Excel) format.
  • You must have share access to both the file (and its enclosing directories) and the HEAT server receiving the data.
  • The data goes into the database as new records.
