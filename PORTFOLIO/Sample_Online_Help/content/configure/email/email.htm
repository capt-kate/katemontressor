Configure > Email

Setting Up Email

Users can create incidents and other business objects (such as problems, changes, and knowledge articles) by email. By default, an object to create an incident using email comes with HEAT. Otherwise, you must create an inbox for each business object and you must provide users with a template.

The configuration for an email service provider is stored as a service business object in the HEAT management database, while tenant-specific configurations are stored within each tenant database.

The email service sends emails using the individual SMTP configuration for the tenant.
