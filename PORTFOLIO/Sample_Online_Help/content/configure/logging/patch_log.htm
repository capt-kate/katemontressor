Patch Log

The Patch log lists information about HEAT development packages. See Using the HEAT Development Project for complete information about HEAT development packages and how to use them.

Viewing the Workspace
  • Within the Service Desk Console, open the Patch Log workspace. The system displays a list of logs.

--or --

  • From the Configuration Console, from the Settings menu, click Monitor > Patch Logs. The system displays a list of logs.

 

Each entry in the Patch log contains the following information:

Field Description
Patch Time The date and time when the patch was created.
Package Name The name of the package.
Index Not used. Defaults to 1.
Patch Name The name of the package. By default, each package has only one patch associated with it, so the patch name is always the same as the package name.
Start Time The date and time when the package started to be applied.
Finish Time The date and time when the package finished being applied.
Errors The number of errors encountered when working with the package. To see the error message, double-click on an entry.
Warnings The number of warnings encountered when working with the package. To see the warning message, double-click on an entry.
