When Seamless Sign-on Fails

A user writes:

I added open ID as an authorization provider to set up seamless sign on. Today, when I try to log in, I it flashes the login screen and then I get a Google open ID authentication request error, unregistered domain. Is there a way to fix this? I can’t log in to turn it off.

Issue

When you login the first time, Google asks if you want to trust the FrontRange Application. If you say Yes, it proceeds. If you say no, you are denied each time.

Solution

Go to the internal user login page when the system is configured to use SSO as a default provider. The URL should be similar to the following:

https://frs-dev-tenant2.saasitdev.com/Login.aspx?NoDefaultProvider=True

  • Clear the cache.
  • Use an internal user account and do unselect on the default authentication provider page.
