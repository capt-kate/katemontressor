Using Mobile

Creating Records in the Mobile Client

By default, forms on a mobile device are read-only. However, administrators can give users the right to add new records.

Log into the Mobile Console and select a role.

If your role has permission to create new records, a + sign on the header appears. If this is missing, your role does not have permission to create new records.

Mobile - Click + to Add an Entry

Editing Forms in the Mobile Client

By default, forms on a mobile device are read-only. However, administrators can assign edit access to designated roles. If your role has edit permissions, you will see an Edit button. You can modify data in:

  • Validated fields (including constrained lists)
  • Date fields
  • Text fields
  • Numeric fields
  • Boolean/bit fields
Quick Actions in the Mobile Client

You can use quick actions related to the business object on the Mobile Client.

  • Click Quick Action in the bottom right corner of the screen.
Saving Records in the Mobile Client

Upon saving the record, the required field list is calculated. The object state may affect the required condition of a field.

  • All required fields must contain a valid entry, or an error message is displayed, and the save aborted.
  • When the validation occurs, the object data is passed to the server.
Sending Emails through the Mobile Client

You can send an email from within the mobile client.

1. While in a record, click Email. The email window appears.
2. Send the email as you would normally.
