HEAT Discovery Asset Management

The HEAT Discovery page enables you to deploy Discovery in various ways.

Access Asset Management
1. From the Configuration Console Settings pane under Extend, click HEAT Discovery.

The HEAT Discovery installers appear.

The HEAT Discovery page

2. Click the name of an installer to begin the installation.
Windows MSI - Administrative Deployment Option

This package enables easy mass deployment of the client via software distribution tools such as DSM, SCCM or Active Directory Policies. Using this method, the Discovery Client is automatically installed to all clients logging into the domain. Their inventory data is then sent to the HEAT platform.

When you are installing the Gateway, you need to provide the following information:

  • Name of the Gateway (this is usually populated automatically).
  • Name of the domain on which the Gateway will install clients.
  • Any associated subnets.
  • The local path of the HEAT Agent Deployment Share.
  • The account name and password on the computers the Gateway will deploy to (the account must have administrative privileges on the client computers).
  • The Client Access Key. The HEAT Gateway Installer Package contains a unique Client Access Key used to authenticate incoming data from the customer. (This is provided automatically during the installation.)

The permissions to the HEAT Agent Deployment Share are set to allow Domain Computers and Users read access.

1. Optional. Check Associate with an OU upon installation, then choose the OU from the drop-down list. The OU will automatically be associated upon installation.
When specifying a default OU at the time of installing the gateway, ensure the computer that you are using to install the gateway does not already have a CI record in HEAT.
2. Click the title to download the MSIPackage file to your local machine. The download confirmation window appears.
3. Choose Save File to place it into your preferred directory, then click OK. The file is downloaded.
4. Extract the zip file and follow the instructions in the enclosed readme file.
Run Windows Client Agent Installer

This package enables you to install client agents on remote (not consistently connected to your intranet) computers running Microsoft Windows.

The client agent installation requires local administrator permissions.

1. Optional. Check Associate with an OU upon installation, then choose the OU from the drop-down list. The OU will automatically be associated upon installation.
When specifying a default OU at the time of installing the gateway, ensure the computer that you are using to install the gateway does not already have a CI record in HEAT.
2. Click the title to download the ClientSetup file to your local machine. The download confirmation window appears.
3. Choose Save File to place it into your preferred directory, then click OK. The file is downloaded.
4. Provide the file to your remote users (generally via email). The users execute the file from their computers.

About NetScanner

The NetScanner automatically runs with the Windows Gateway Datacenter Edition installer. NetScanner will return only IP addresses and names of devices without clients installed. NetScanner has the following features:

  • Supports SNMP V1, V2C, and V3. Users can configure multiple protocols.
  • Able to discover routers and switches.
  • Able to configure known devices as defined in the NetScan.xml file.
