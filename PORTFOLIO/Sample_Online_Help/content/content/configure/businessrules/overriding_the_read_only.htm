Overriding the Read-Only Rule

You can override the read-only business rule.

Overriding the Read-Only Rule from a Quick Action
1. From the applicable business object workspace, click Action Menu > Edit Actions to open the quick actions center.
2. Click the Create New Actions tab on the left and choose Update Object.
3. In the Field Name field, choose ReadOnly. In the Field Value field, enter True.

Update Object Quick Action - ReadOnly Field

4. Enter information into the other fields as needed.
5. Click Save.

Users can choose this quick action to apply to the read-only field on a specific business object record if the read-only field is set up for that business object.

Overriding the Read-Only Rule from the Role Definitions
1. From the Configuration Console Main Menu, choose Users and Permissions > Roles and Permissions.
2. From the Role Name column, choose the role for which you want to modify the read-only rule. The Role Details page appears.
3. Choose the Object Permissions tab.
4. Find the object that you want to override for this role, then click Edit under the Lifecycle column.
5. Under When Business Object is in Final State, check Allow Editing.
6. Click Save.
