Troubleshooting Data Integration

The fastest way to spot errors is to look at the integration log. See View the Data Integration Log - File Integration and View the Integration Log - SCCM on how to use this feature.

While the job is being processed, you can view the record from the Integration Queue workspace. View the Integration Queue.

Import All Data Regardless of Last Audit Date

Q. I've had to make some attribute mapping changes to the SCCM integration of a customer. Whenever I run the integration, it appears to be pulling information for all computers which have been audited since the last SCCM integration was run.

Does anyone know if it is possible to perform a complete import of all computers regardless as to when they were last audited by SCCM?

A. Within the Integration workspace, set the last scanned date to a really old date, such as January 1, 1949.

Integration Workspace

Let the job run as scheduled and all information will be captured.
