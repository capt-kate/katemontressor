Text is Double-Spaced Using Copy and Paste

When you copy and paste an email from Outlook into a multi-line HTML field, the lines become double-spaced. This occurs because Outlook inserts a non-breaking space (&nbsp;) into each empty line. This issue is also browser dependent, as it occurs when using Firefox or Chrome, but not Internet Explorer.

For example, within Outlook, you can view the source text of your email (right-click > View Source) and observe the extra code inserted, such as:

<p class="MsoNormal"><span style="color:#1F497D;mso-ansi-language: EN-US" lang="EN-US">&nbsp;</span></p>

This code does not appear in the text of the email, but it interpreted correctly when pasted into HEAT.

To remove the extra lines, delete them manually.
