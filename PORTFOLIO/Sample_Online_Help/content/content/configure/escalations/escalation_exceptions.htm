Escalation Exceptions

You can define exceptions to the escalation lifecycle based on field values within the escalation. You can next exceptions within other exceptions.

For example, the Incident Waiting escalation is based on the incident status field.  It sends a reminder email to the customer after 14 days and after 28 days, it closes the incident if the customer does not respond to the request for information.

Incident Waiting (Without Exceptions)

To define an exception based on the Priority field so that the incident remains open for a longer time if it has a priority of 1, modify the exception priority. In this example, select the Priority field and a value of 1.

The exception is further configured so that:

  • Priority 1 incidents remain open for 40 days instead of 28 days.
  • For Priority 1 incidents, the system sends the first reminder email to the customer after 6 days instead of 14 days.

Define an Exception
1. From the Escalation Schedules page, select a schedule.
2. Click Edit Schedule. The Edit Escalation Settings window appears.

Edit Escalation Settings Window

3. Select a field from the field drop-down list. The available fields correspond to the associated business object.
4. If viable, the Value area shows each member of the pick list associated with this field. Select one or more options to select pick list items.
5. Click Save to save the exception. The system changes the escalation schedule to display each selected pick list item listed as an individual exception beneath the parent field.
6. To delete an exception from the database, select the exception and click Delete Exceptions.
Extend Exceptions

You can extend an exception with additional exception settings. To do so, select an exception from the list and modify the exception, selecting additional fields and values to build out very specific exception circumstances.

Set Exception Thresholds

You can customize exceptions by adding specific threshold settings as needed, or by populating the exception with thresholds inherited from the parent schedule.

To set additional exception thresholds, do the following:

1. From the Escalation Schedules page, click Set Thresholds. The Escalation Parameters for window appears.
2. Enter information into the fields.
-fields-
Field Description
Schedule is enabled Makes the escalation schedule active upon save. You can now edit the fields.
Target Duration Defines the interval used to track this escalation by a specific time in days, hours, minutes.
Target DateTime Field Uses a field for the target schedule.
Hours of Operation Set the hours of operation. See Setting Up Hours of Operation.
On breach, run action The quick action to run if the escalation lifecycle exceeds the time defined.
3. Edit the settings on the Escalation Level 1, 2 and 3 tabs as needed.
4. Select OK to save the exception. The system changes the escalation schedule to show each selected pick list item, listed as an additional exception tier beneath the parent field.
