* * * New or Improved for 2014.2 * * *  Using Grids

A grid shows multiple records in a table view. Columns represent the fields for each record. Grids are defined within each business object, and are then viewed in a layoutA collection of forms, tabs, and grids that shows the parent and child records of an object. You can access a detailed view of a the parent record..

Administrators can customize the colors of a grid within the Grid Highlighter.

View the Grid List
  • From the Details page of a business object, click Grids. A list of grids associated with this business object appears.

Sample Incident Grid List

Create a Grid
1. From the Configuration Console Settings pane, click Build > Business Objects, then select the business object to which you want to create a grid. The business object details page appears.
2. Select the Grids tab. A list of grids created for the business object appears.
3. From the top of the page, click Add New Grid. The grid editor appears.
4. In the Grid Definition Name field, enter a unique name for this grid .
5. From the Available Fields panel, expand the Fields list, then drag and drop each field you want onto the Column Properties pane. Each field appears as a column in your grid. When your cursor is in the right place to drop the field, a green check mark appears. You can drop the fields in any order and reorder them later (by dragging them around).

Green Checkmark Denotes Drop Area

As you add each field, a column header appears in the Grid Preview pane.

6. Define the properties of each grid column as needed.
-properties-
Property Description
Display Name Adds a customized column name. Double-click the field and enter the custom text.
Field Reference

The name of a valid business object field.

You also can use the following references to include data from related business objects:

  • Use of a link field. For example, [link:OrgUnitLink]Name
  • Combining a business object type and internal relationship name in relationship instances using a link_RecID field. For example: [OrganizationalUnit#.]Name
Renderer A rendering option. Select from the drop-down list. For example, you can render the column as an option to be checked, or as a Yes/No option. These options are different for each field type. See Field Renderer Options below.
Style Expression Optional. The style name created in Grid Highlights
Width The pixel width of this column (default: 100).
Sortable Enables sorting features for this grid column. The column name is added to the Sort Column drop-down list.
Groupable Enables grouping features for this column. The column name is added to the Group By drop-down list
Wordwrap Enables word wrap for column text. By default, all column text is displayed as one line, allowing longer entries to be obscured in the grid.

7. In the Properties pane, define grid settings.
-grid settings-
Setting Description
Default Sort Column Optional. The column by which the records will be initially sorted. Select from the drop-down list. Only columns set as Sortable are included in this list.
Default Sort Order The order in which you want the records to be sorted: ascending or descending.
Default Group By Column Optional. The column by which the records will be initially grouped. Select from the drop-down list. Only columns set as Groupable are included in this list.
Column to Display as Link Optional. Shows the values as links to other workspaces. For example, if you have a grid listing incident records, you could have the Owner column shown as a link (blue color text underlined). The link action is defined in Link Configuration (below).

8. Optional. In the Other Settings pane, add formatting rules and configure your link fields. See Formatting Rules and Link Configuration
9. Click Save Design .
Link Configuration

If you designated a Column to Display as Link, you must configure what happens when the user clicks the link. Including a link field in a grid can connect a grid column entry to its parent object. For example, a line-item cost can be connected to a parent object such as its related configuration item.

1. Click Link Configuration. The Link Configuration window appears.
2. Choose a link action from the drop-down list. This action occurs when the user clicks the link.
Action Description
Go to this object Goes to the parent business object.
Go to linkfield Goes to the linked field defined in the Linkfield name field.
Linkfield name If you chose Go to linkfield, enter the name of the linked field.
3. Click OK.
Duplicate a Grid
1. From the Grid list, move your mouse pointer over a grid name and notice that the Copy Grid icon appears.

Copy Grid

2. Click Copy Grid , enter a new grid name, then click Save.
3. Click Save from the toolbar.

A copy of the grid with the new name appends to the existing grids.

Modify a Grid
1. Click to open the grid that you want to modify. The grid editor appears.
2. Change the information as needed. See Create a Grid above.
3. Click Save from the toolbar.
Delete a Grid
1. From the Grid list, move your mouse pointer over a grid name and notice that the Delete Grid option appears.

Delete Grid

2. Click Delete Grid.
3. Click Save from the toolbar.

The page is refreshed and the grid is deleted from the business object.
