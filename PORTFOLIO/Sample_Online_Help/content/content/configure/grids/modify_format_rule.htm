Formatting Rules

Formatting rules are lines and colors that are used in grids. For example, you can show grids with alternating yellow rows or you can show grids with lines (borders) between each row.

You can define a style to use within the grid highlighter (see Grid Highlights), or you can create an expression specific to a single grid to define your colors and conditions.

After you save the formatting rule and apply it to a grid, you can modify the expression to accommodate any changes.

1. Within the Configuration Console, open the business object to which you want to use a grid style.
2. Choose the Grids tab. The list of grids created for that business object appears.
3. Click the name of the grid you want. The grid editor appears.
4. Click Formatting Rules. The list of formatting rules appears, if any.
5. Click Add Formatting Rule. The Add Formatting Rule window appears.

Add Formatting Rule Window

6. Enter your information into the fields.
Field Description
Apply to Row

Applies the rule to the entire row (as opposed to a column).

Note: You cannot use this option with the Column Name option.

Column Name

To use the rule for a column, select a column name from the drop-down list.

Note: You cannot use this option with the Apply to Row option.

Because the style is usually conditional, this option results in applying the style to single cells.
Style Expression

Double-click a defined style from the right pane styles list. The style name appears in the text area. (See above image.)

OR

Enter an expression that defines the style that you want. For example, the expression $(iif((Status == "Closed"), "RedStyle", "LimeStyle")) returns the row as red if the record status is closed and returns the row as lime green if it is not closed.

You can choose fields, functions, and styles from the tree on the right to use in your expression.
7. Click Save, then click Save again.
8. In the grid form, click Save Design to apply the updated rule.
