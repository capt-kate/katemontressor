Assigning Layouts (Workspaces) to Roles

You can create a unique look for each role.

View Which Layouts are Available to Each Role
1. From the Configuration Console Settings pane, click Build > Page Layouts. The system displays the list of roles and page layouts.

You can define which layouts (workspaces) each role sees.

2. Select a role from the list. The available workspaces appear in the Page Layouts column.

Page Layouts

Add Layouts to a Role

Roles see each layout as a workspace. Each workspace is represented as a tab in the top header bar.

1. On the Page Layouts page, select the role that you want, then click Add Layout. The list of workspaces (layouts) appears.
2. Click the layout that you want. The Tab Details page appears.
3. Select or enter the options that you want. See Create a Layout.
4. Click Save. The layout is now visible to everyone in that role. Users might need to refresh their screen to see the updated options.
Modify a Layout

You can modify layout options from the Page Layouts page, as well as within the business object Layouts tab.

  • From the Page Layouts page, select the role that you want, then next to the layout that you want to modify, click Edit. The layout editor appears. See Create a Layout.
Modify Tab Details

You can modify the availability and visibility of the tab for each role.

  • From the Page Layouts page, select the role that you want, then next to the layout that you want to modify, click Advanced Options. The Tab Details page appears. See Making Layouts Available to Users.
Remove a Tab From a Role

Removing a tab from a role does not delete it from the business object.

1. From the Page Layouts page, select the role that you want, then next to the layout that you want to modify, click Delete. The delete confirmation window appears.
2. Click Yes to confirm the delete. The layout no longer appears to the user role. Users might need to log and log in again to see any changes.
