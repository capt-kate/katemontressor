The Incident Management Wizard

The Incident Management wizard is part of the Application Setup Page. Use this wizard to define how your Service Desk will manage incidents.

Process Wizard: Incident Management

Services Provided

Define the Services that are supported.

1. Click Add Service.
2. Enter the Name and Description of the Service.
3. Click Save & Activate.

The category is added to the list and rendered active. By default, each Service is active and available for customization.

Click the Edit link for an existing Service to edit its name or description.

Classification

Define incident categories for each service. Classifying incidents aids you in assignment, escalation, problem analysis and reporting.

Select a Service to view all the Categories that are associated with the Service.

  • Click Add Category to display a window for entering the name of a Service category. Upon clicking Save & Activate , the category is added to the list and rendered active. By default, each Service Category is active and available for customization.
  • Click Deactivate for an existing Service Category to disable it. Click Activate to enable it again.
  • Click Rename for an existing Service Category to rename it. Enter a new name for the Category, then Save.

Escalation

Specify the response, type of response and resolution schedule based on classification, service level and priorities.

Use this module to define how your service desk will prioritize and respond to incidents in the HEAT Escalation Engine. This includes the total resolution time in an escalation lifecycle upon its creation, and the length and dependencies of each escalation stage.

Factors affecting escalation schedules can include hours of operation and service level agreements. Throughout the incident Management process, users are notified when there is a change in the status of the incident and when the incident is resolved.

By default, the wizard displays Categories 1-5, each with its default time limits for responding to an incident once logged, as well as a default interval for resolving the incident.

To customize these settings for a Priority level, click its Edit link.

1. Click the Target Resolution time checkbox to enable this portion of the escalation schedule. Here, you define a target resolution time for the incident. This can be dictated by a Service Level Agreement.
  • Enter a number value in the field provided.
  • Select an interval unit from the drop-down list (Minutes, Hours, days).
2. Select the Hours of Operation for your organization.
3. Set the levels of escalation associated with the resolution cycle.
  • Click the After checkbox for each escalation level to enable. You can define up to three escalation levels.
  • Enter a number value in the field provided.
  • Select an interval unit from the drop-down list (Minutes, Hours, days).
  • From the drop-down list, select an action to be taken when the time limit has expired on the escalation.
  • From the After the target time has reached drop-down list, select an action to be taken when the time expires on the resolution cycle.

For example, a Priority 1 incident is set to resolve in 1 day. The first escalation is set to occur after 5 hours and 2 minutes (21%) from the incident creation time. The second escalation is set to after 7 hours and 12 minutes (30%) from the incident creation time. The third escalation is set to occur after 19 hours and 55 minutes (83%) from the incident creation time. A notification is sent to the Service Desk Manager when the resolution time is breached.

4. Indicate the action that will occur when the target time has been breached from the After the target time has been reached drop-down list. Select an action to be triggered.

Incident Lifecycle

Select the actions to take at each stage of your incident management cycle. If you have advanced lifecycle requirements, you can define them later. The default lifecycle settings are:

  • Logged or requested
  • Active
  • Waiting for customer
  • Resolved
  • Closed
1. Select if an action should occur at each stage of the incident's lifecycle.

For example, by default, when an incident is logged or requested, the incident record is added to the My Items tab in Self Service. By default, when an incident is closed, the cost of the incident is set to update and a survey is sent to the requester.

2. Select an escalation's Send Email checkbox to automatically send an email notification when the incident is updated to the matching status.
  • Use the associated drop-down list to select a an email category, such IncidentClosureNotification when the incident is in Closed Status.
  • Click the Preview Email link to preview the message.
3. Click Add another Action to select an additional action.
  • Select an Action Type.

For example, by default, the only action that occurs when an incident is in waiting for customer status is the StopEscalationClock action. You could add a SendEmail action type here.

  • Select an Action.

For example, for the Waiting for Customer Status, along with the SendEmail qction type, you could select the waiting for customer reminder action.

4. Click Save.
Teams

Identify the teams that your organization will use to support its service desk.

  • Click the Add Team button to add a team.
  • Enter a unique name for the team in the Title field.
  • Enter the email address for the team in the Email field. You can create a unique email account for each team.
  • Select Service Desk to specify if this team is included in the Service Desk support for first line incident response and resolution.
  • Click Save.

Active teams are designated by a Service Desk: Yes entry. To remove a team from the list of available Service Desk support options, click its Deactivate link. Its entry changes to Service Desk: No.

Review and Finish

Review the items to determine their accuracy. You can return to any of the pages in this wizard to edit these wizard components.

Click Finish to save these changes to the database.
