Using Task Status with WaitForChild

This example shows how to set up a workflow that waits for multiple tasks to be Completed before continuing.

Create a Workflow
1. Within the Configuration Console, open the Workflows page, then click Add New .... The Add New Workflow page appears.
2. Enter the parameters.

Add New Workflow Page

-parameters-
Parameter Description
Business Object Choose a business object from the drop-down list. In this example, choose Incident.
List Name Enter a unique name for the workflow object. In this example, enter IncidentTaskStatus.
Display Name Enter a name that appears on workspaces. In this example, enter IncidentTaskStatus.
Description Type a brief description of what the workflow does.
3. Click Save. The Add Workflow page appears.

Add Workflow Page

4. Click Add Version. The Workflow Designer appears.
5. Drag Start, Wait For Child, and Stop onto the editor, and connect them.

Workflow Designer

6. Double-click the header on Wait For Child. The properties window appears.
7. Enter the following properties.
-properties-
Property Description
Title IncidentTaskStatus
Relationship Name IncidentAssocTaskAssignment
How many children All
Child Conditions AND
Timeout Duration: 5 minutes
Field Status
Operator Equal To
Value Completed
8. Click Save. The workflow now appears on the workflow list.

Workflow List

Test the Workflow

Test the workflow by creating an incident record with multiple tasks. Set the tasks to various states and observe the Workflow Instance.
