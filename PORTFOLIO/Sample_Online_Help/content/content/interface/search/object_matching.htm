Using Object Matching

Object matching is a type of search that allows you to search other business objects while inside the current form. The search can originate in two ways:

  • The search can originate from a field on a form that has an associated search icon. An example is the Summary field on the Details tab for an incident.
  • The search can originate when you enter a search phrase into the Search text box in the search results panel and start the search.

To start the object matching search, click the Search icon associated with a field or use the Search text box in the search results panel. Objects that match the search criteria are ranked by how closely they match the current object. These matching objects are listed in the search results panel in the form of the current object.

The administrator defines which fields can perform object matching.

For example, you can click Search next to a field to search for other business objects that contain any of the words anywhere in the record. If you highlight specific words, the search uses only those words for the search.

The business objects search results are set by default for incident, knowledge base, and problem. However, the administrator can modify them.

When objects are searched, object matching performs a Microsoft SQL full-text search in all full-text indexed fields in the object tables. (Examples of objects are incidents and problems.) The Microsoft SQL engine also ranks the objects.

Run an Object Matching Search

Use object matching when you create a record by quickly searching the configured field to compare it to all other records. (Examples of a record are an incident or problem.) You can perform this search on an existing record to search for duplicates.

Only one field can be configured to begin object matching. The administrator defines that field.
1. Open a record or create one (such as an incident or problem).
2. Run an object matching search in one of two ways:
  • In the open record, use the text from the field next to Search . When you start your search from here, the text in the field automatically populates the search results text box.
  • In the search results text box, enter the text to use for the search.
3. Click Search to start the search.

The results that match your search requirements appear in the Related Items panel.

The search results are sorted according to the best match. If you enter multiple words in the search results text box, the search uses the OR phrase by default and includes results that use any of the words. The search results are ranked based on the frequency of word occurrence and the number of matching words that are found in the results. Items closest to the top of the list typically represent the best matches based on your search criteria.

View the Object Matching Search Results

The search results from object matching appear in the Related Items panel. The search results are sorted according to object. Placing the mouse cursor over a result highlights it and displays a drop-down arrow. If you click the drop-down arrow, a list of actions expands under the selected search result.

The administrator can configure the number of matches shown per object (incidents, problems, and knowledge by default). The length of the preview text for each match also can be configured. For example, the administrator can configure the preview text to show the top 5 or 10 records for each category and 2-3 lines of text preview for each match.

  • Open Preview: Opens the matching record in a preview window. The preview window contains links to open the record in a new or active tab. The preview window contains additional actions depending on the record type.
  • Open in New Tab: Opens the full matching record in a new tab.
  • Open in Active Tab: Opens the full matching record in the same tab.

Other actions might appear as configured by the administrator.
