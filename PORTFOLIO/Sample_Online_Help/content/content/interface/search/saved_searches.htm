* * * New or Improved for 2014.2 * * *  Using Saved Searches

You can save your search criteria in a saved search. By saving searches, you can quickly perform a search task without the need to manually enter all of the search parameters each time. You can create, modify, and delete a saved search.

The Saved Searches link is located on the left side of the search area in the upper-right side of a form.

Saved Searches Link

Click Favorite Searches for a list of all searches tagged as favorites (either by default or by another user in your role).

Run a Saved Search

You can run a search from your favorite search list or from all the saved searches that are available to your role.

Run a Search from the Saved Search List

1. Click Saved Searches on the search bar. A list of saved searches appear.
2. Choose a search from the list. The search runs immediately and the results of the search appear.

Run a Search from the Favorite Searches List

Click the drop-down arrow in the Favorite Searches text box on the search bar, then select a search from the list. The search runs immediately and the results of the search appear.

Create a Saved Search

HEAT provides an assortment of default saved searches according to your user role. You can also create your own search and add it to the saved searches list. For a demonstration on how this works, see Video: Creating a Saved Search.

You can only create saved searches for personal use if your role has create permissions. If your role has edit permissions, you can edit saved searches created by other users and publish saved searches to other roles. Permissions for saved searches are defined by the administrator in the system settings for each role.
1. Within a workspace, click Saved Searches from the search area. A list of saved searched appears.
2. Click New Search. A blank search form appears.

Saved Search Form

The New Search button only appears if your role has create or edit permissions.
3. Enter information into the fields.
-fields-
Field Description
Enter Search name here Enter a unique, descriptive name for your search, then click Save.
Search Favorite Click to add this saved search to your favorites list. If this search is not published to any role, it is indicated as your personal search by a blue star. If the search is published, it appears on the saved search menu for this role and is indicated by a yellow star.
Publish action to Choose the roles you want from the drop-down list. This saved search is available for all users in those roles.
Category Select if the search is on the saved search list or on a dashboard. Category is a way to group searches into meaningful collections.
My Default Click make default if you want this saved search as the default when you (the logged-in user) open this specific workspace. This search automatically executes and the results appear in the grid when you open this workspace.

Click remove default if you do not want this to be your default search.
Default for roles Choose the roles from the drop-down list if you want to make this search a default. You can only designate roles to which you have write permissions.
If you have create permissions, you can only publish to the roles you belong to and the option to publish to other roles are grayed out. If you have edit permissions, you can publish to other roles.
4. Enter or choose your search expressions.

Search Expressions

a. Business object Choose a business object from the drop-down list.
b. Field Choose a field from the drop-down list. The available fields depend on the business object that you selected.
c. Operator Choose an operator from the drop-down list. Various options appear based on the field that you selected.
  • Equal to: The value in Field matches (=) the value in Criteria.
  • Not Equal to: The value in Field does not match (≠) the value in Criteria.
  • In List: The value in Field is a value from the list chosen in Criteria. You can choose multiple lists.
  • Not in List: The value in Field is not a value from the list chosen in Criteria. You can choose multiple lists.
  • Greater than: The date or numeric value in Field is greater than (>) (but not equal to) the value in Criteria.
  • Less than: The date or numeric value in Field is less than (<) (but not equal to) the value in Criteria.
  • Greater or Equal: The date or numeric value in Field is greater than or equal to (≥) the value in Criteria.
  • Less or Equal: The date or numeric value in Field is less than or equal to (≤) the value in Criteria.
  • Begin With: The value in Field starts with the string value in Criteria (ie. "ac" would find "action", "activate", "accessory", etc.
  • Not Begin with: The value in Field does NOT start with the string value in Criteria.
  • Contains: The value in Field contains the string value in Criteria.
  • Does Not Contain: The value in Field does not contain the string value in Criteria.
  • Is Empty: Field does not contain any values.
  • Is Not Empty: Field contains something.
  • In Range: The value in Field is within the range of the value in Criteria. For example, HoursOfOperation uses a range of hours, and Urgency uses a range of classification.
  • Not in Range: The value in Field is not within the range of the value in Criteria.
  • Full text search: This is similar to a Google-type keyword search on Field, rather than an exact match of a fragment of the word. For example, if you are looking for "run", the results include "run", "ran", "running", etc. For an exact match of "run", use Contains.
  • Full text not search: This operates the same as full text search, but excludes matches containing the value in Field.
d. Criteria Enter a value to search.
  • For a text value, enter the text.
  • For a date/time field, click the Calendar picker
    OR
    if the operator is equal, type $ to view a list of date/time functions.

For a validated field and the operator is in List, choose the options that you want from the drop-down list. You can select multiple options.

You can also enter an expression. For example:

$(Prompt("Please Select Priority","Priority of IncidentPriority used in Incident",200,400))

See Using Expressions in Searches and Dashboards for more information.
5. Click Add to add more search parameters.
  • The AND operator finds a record if both conditions are true.
  • The OR operator finds a record if either condition is true.

To delete a line of search criteria, click Delete .

6. Click Preview to view the search results based on the search criteria.
7. Click Save and Close. The search is now saved and appears on your list of saved searches.
Saved Search Dependency

Saved searches that are deleted or modified might affect the display of data in a dashboard part. You can view the dependencies associated with each saved search to determine if a dashboard part is affected.

View the Dependencies of a Saved Search

1. From the workspace, click Saved Searches. A list of saved searches available to your role appears.
2. Hover your cursor over the search name. If you have the correct permissions, several links appear: edit, delete, dependency.
3. Click Dependency. An informational window appears.
4. Click OK to dismiss the window.
Modify a Saved Search
  • Do not modify any default saved searches, as this could cause unexpected results due to dependent processes.
  • Before modifying a saved search, check for any affected dependencies.

Any saved search can be modified or published to any role if you have edit permissions. In this case, an Edit button appears next to the saved search. Permissions are set by your administrator in the system settings for your role.

1. From the workspace in which you want to modify a saved search, click Saved Searches. The list of saved searches available to your role appears.
2. Hover your cursor over a search name. If you have correct permissions, an Edit button appears.

Edit a Saved Search

3. Click Edit , then change any of the fields and search criteria.
4. Click Save.
Saved Searches and the Favorites List

The saved searches marked as favorites appear on the saved searches list. Favorites are established globally for all users by the administrator, but individual users can manage their own list of favorites. If you want a saved search that is not on the list, you can add it. You can remove a search from the favorites list that you do not want on your favorites list without deleting it.

Remove a Saved Search from the Menu

1. Select the workspace for which you want to remove a saved search.
2. Click Saved Searches. The list of saved searches available to your role appears.
3. Click Edit on the same row as the saved search that you want to remove.
4. Deselect Add to Search Favorites.

Add a Saved Search to the Menu

1. Select the workspace for which you want to add a saved search.
2. Click Saved Searches. The list of saved searches available to your role appears.
3. Click Edit on the same row as the saved search you want to add.
4. Select Add to Search Favorites.
Delete a Saved Search

Any saved search can be deleted if you have delete permissions. In this case, a Delete button or link appears next to the saved search. Permissions are set by your administrator in the system settings for your role.

Before deleting a saved search, check for any affected dependencies. See Saved Search Dependency.
1. From the workspace in which you want to delete a saved search, click Saved Searches.
2. Hover your cursor over a search name. If you have the correct permissions, a Delete button appears.

Delete a Saved Search

3. Click Delete . The Delete confirmation window appears.
4. Click Yes.
Access Saved Searches by Business Object

Administrators can see all saved searches for each business object.

1. From the Configuration Console Settings pane, click Build > Automation Tools > Saved Searches. The list of business objects that contain saved searches appears.
2. Click Edit under the Action column for the business object for which you want to view saved searches. The list of saved searches for the business object appears.
3. Hover over an item from the list. The Edit button appears.

The Saved Search Edit Button

4. Click Edit . The Saved Search Edit page appears.
5. Modify the saved search.
6. Click Save & Close.
