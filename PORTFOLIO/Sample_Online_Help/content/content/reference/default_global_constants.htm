SLAEnabled

 

False

Text

Enable or disable SLA driven escalation for incident and service request.

The SLA calculation is based on this global constant by default.

If the value is False (default), the SLA will not be associated for incidents and service requests that have been created.

If the value is True, the SLA will be associated for the incidents and service requests.

This behavior is introduced in HEAT Release 2013.1 and differs from the behavior of earlier versions. This change was introduced, to simplify the escalation model in effect for the incident and service request.
