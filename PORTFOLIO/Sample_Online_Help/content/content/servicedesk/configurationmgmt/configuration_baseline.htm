Configuration Baseline

A configuration baseline shows the configuration of a CI that has been reviewed and agreed upon and is marked to be under change control.

Only the CI and CIComponent business objects can be baselined.

Using the configuration baseline, the Configuration Manager can view and compare all historical changes made to the CI and the related CIs. The Configuration Manager can view the current baseline against the historical baseline.

Establishing a baseline is useful in case it is necessary to back out after a change and restore the original configuration, or if it is necessary to rebuild a specific version of the CI at a later date. If the CI is linked to a change, the baseline automatically runs when the change is in implemented status.

To baseline a CI, the CI type must be under change control by the administrator. In addition, you must

  • Select properties of the CI type to baseline.
  • Select the change control option on the Details tab of the CI record.
Select Properties to Baseline

Before recording a baseline action, ensure the fields of the CI type that you want to record for baseline are selected.

1. Log into the Service Desk Console as the Configuration Manager or administrator, and open the Baseline Mapping workspace. A list of CI types and components appears.

Baseline Mapping Workspace

To view and compare baselines of a CI, the Under Change Control option for that CI type must be selected by the administrator for the CI business object. If the Under Change Control option for a CI type has not been selected by the administrator, you will not see that CI type listed.

2. From the CI types (or CI Components) list, select a CI type (or CI component). The suggested fields to baseline for the selected CI type (or component) appear.

CI Type

3. To view all the fields of the selected CI, click Show All Properties. Select the properties that you want to baseline.
4. To baseline all CIs related to the selected CI, click Show Related CI and CI Components, then select the checkboxes next to the ones that you want to baseline.
5. Click Save .
View the Configuration Baseline

To view and compare baselines of a CI, the Under Change Control option for that CI type must be selected by the administrator.

Within a CI record, select the Details tab, then select Is Under Change Control.

CI Record Details Tab

If you do not see the Under Change Control option, then it has not been selected in the Configuration Console by an administrator.

Running the Run Baseline quick action for a CI type while the Under Change Control option is not selected generates an error log within the Baseline Comparison tab of the record.

View a Configuration Baseline and Compare Baselines

1. Within a CI record, select the Is Under Change Control option on the Details tab and Save.
You can only view the baseline of a CI type for which the Under Change Control option has been selected by the administrator. Also, ensure that the Change Control option is selected on the Details tab of the CI record. If the CI type business object does not have the Under Change Control option selected in the Configuration Console, you will not see the Change Control option on a record of that CI type.

CI Record Is Under Change Control

2. From the toolbar, select Run Baseline.

You can also run the baseline from under the Baseline Comparison tab or from the Run Baseline quick action menu.

 The Baseline Creation window appears.

3. Enter a baseline name and description in the Name and Description fields, then click OK. The baseline might take a few minutes to run.

The current baseline is recorded.

4. Click OK at any time.
5. View the baseline history from the Baseline Comparison tab.The baseline history table shows the list of all the baselines for the CI at various times. The last successful baseline is the baseline, while the rest are historical data.

Double-click any row to see details.

Baseline History

  • Click Display Baseline to show the values of the fields that have been selected to show baseline information.
  • Click Create Baseline again to run another baseline if a change has been made to this CI.
  • Click Show Variance from Baseline if you made any changes to the current baseline.
  • Select the baselines from the history list, then click Compare Baseline/History to compare baselines. The properties that have changed appears.

If an error occurs, the baseline will not run and the error appears in the Error Information section below the baseline history table.

If the CI is related to a change, the baseline automatically runs when the change status is implemented. If necessary, the baseline can be run at any time by clicking the Run Baseline quick action from the Change workspace.
