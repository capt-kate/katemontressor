Variance Reports

You can view a variance report to see all CIs that have been updated. The report shows as a dashboard part in the Configuration Manager dashboards.

To baseline a CI, the CI type must be under change control by the administrator. In addition, you must select the properties of the CI type to baseline and select the Change Control option on the Details tab of the CI record.

The two ways in which variance reports are generated are:

  • By running a quick action from the list view in the Configuration Manager workspace.
  • By scheduling the quick action to run at a defined interval.

If you want a configuration item to revert to its last recorded baseline, you can create a Request for Change.

The newly logged change, shows the variance information in its Variance Details tab.

Generate a Variance Report from a Quick Action

You can select a few CI records for which you want to view variance reports, select the Run Variance quick action, and then view the report in the Variance Reports dashboard part.

1. Log into the Service Desk Console as a Configuration Manager and open the CI workspace. A list of configuration items appears.
2. Select the CI records for which you want to view variance reports.
3. From the Action Menu, select Run Variance Report > Run Variance. In the Run Variance Report quick action, the Run Async flag is selected.
4. The report appears in the dashboard.

If there are no changes to the CIs from the last time the report was generated, no records appear in the dashboard part.

Schedule a Variance Report

By default, the Run Variance workflow for the schedule entry business object is available to the administrator for managing the scheduling of the variance report.

To make sure that the scheduler runs effectively, the Run Async flag is unchecked for the quick action Run Variance Through Scheduler. The scheduled variance report uses the saved search CIs in Change Control.

The default variance schedule is stored in the Schedule Entry workspace as WeeklySchedule Run Variance quick action.

View the Variance Report

You can view a variance report to see which CIs have been modified.

1. Log into HEAT in the Configuration Manager role.
2. Make sure that the Configuration Manager dashboard is selected on the Home tab.
3. View the dashboard part called Run Variance.

If there have been no changes in the configuration of the CIs, there are no records displayed in the dashboard part.

Only the CIs that have changed since the last time the baseline was taken appear in the dashboard part.

To view the changes, click the CI name link and view the attributes that have changed in the From Value and To Value columns.

Create a Request for Change from a Variance Report

After viewing the variance report, you can choose to log a Request for Change for the configuration item that is affected.

1. View the variance report.
2. Click the Configuration Item tab.
3. Search for the configuration item by entering its name in the Search field and then click Search.
4. Select and open the record.
5. From the action menu, select No Category > Create RFC for Variance.
6. Click the Change tab of the configuration record.

The change is set to logged status.

7. Open the change record and view the defined field values.
8. Modify the fields, if appropriate.
The change remains in logged status until it is set to requested.
9. Notice that the Variance Details tab on the change record shows the details of the variance from the last baseline.
10. When you are ready for this change to be worked on, select the record and click Go to.
11. Click the Request Change quick action from the toolbar.
