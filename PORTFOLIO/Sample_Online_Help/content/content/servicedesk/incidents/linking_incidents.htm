Linking Assets

While working on an incident, you could determine that the incident is either part of an existing change, problem, release, or other record type. For example, you might notice that several incidents have been logged against the same configuration item. Upon further review, you determine that the incidents are actually part of the same problem.

Although there are no set default restrictions, an incident should have only one record type (such as problem) associated with it. However, some record types might have multiple related incidents (such as problem, change, or knowledge).
  • If there is no existing problem to link to the incident, but you assess that the incident is part of a problem, you can create a new problem to link to the incident. The problem is created in a logged state until it is reviewed and diagnosed.

    •  

    Link an Incident to Another Record
    1. Within the incident record under the Details tab, click Link in the Assets area. The assets window appears.
    2. Find the record you want to link, then click Select. (See Using Record Search.) The item appears in the Assets area.

    Sample Linked Asset

    3. Click Save from the toolbar.
    • The record is linked to the incident. If the linked record is a problem, change, CI, or knowledge article, the item also appears under the corresponding tab in this record.
    • The incident also appears under the incident tab of a linked problem, change, CI, or knowledge record.
    • When a change or problem is set to Closed, the related incident is updated to Resolved.
