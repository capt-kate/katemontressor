Working with the Problem Review Board (PRB)

The goal of the Problem Review Board is to manage problems. This is done through analyzing incidents as they happen, reviewing historic trend data, and staying current with industry news and vendor updates.

The Problem Review Board must include representation from relevant individuals to effectively review incident trend data for reactive Problem Management as well as searching for risks from a proactive stance.

Add a Member to the Problem Review Board
1. Within a problem record, select the Problem Review Board tab. The list of board members appears, if any.

Problem Review Board Workspace

2. Click Link. The employee list appears.
3. Select the employee who you want to add as a member and click Select.

Add Employee to the Problem Review Board

The name appears on the Problem Review Board list.

4. Repeat to add board members.
5. Click Save from the toolbar.

The employees added to the Problem Review Board are notified by email.

Each member should proactively oversee and manage the successful resolution of the problem.

Create a Problem Review Board

Each Problem Review Board contains individual members and is associated with a problem of a specific priority. A notification is sent to each member of the Problem Review Board when:

  • The root cause of a major problem is identified.
  • A major problem is resolved.
  • A priority 1 problem is logged.
  • A service is linked to the problem.

Follow these steps to create a Problem Review Board:

1. Log into the Service Desk Console as an administrator and open the Contact Group workspace. A list of groups appears, if any.
2. Click New Contact Group. A blank Contact Group record appears.
3. Enter information into the fields.
-fields-
Field Description
Group Name A unique name for the group.
Group Type The type of group that you are creating. Choose Problem Review Board from the drop-down list.
Problem Priority The problem priority level that this board should review. Ensure that another Problem Review Board is not linked to the same problem priority, as this could cause unexpected results. If there is already a Problem Review Board for the problem priority that you are selecting, edit the existing record instead of creating a new one.
Change Type

The type of change that the Problem Review Board should review. Choose from:

  • Emergency
  • Major
  • Minor
  • Significant
  • Standard
Status The status of the board. Choose Active or Inactive.
Approval Type

The type of board that you are creating. Select from the drop-down list. This is populated from the TypeOfApproval business object.

Details A description of the function of the board.
4. Under the Member tab, click Link to add board members.
5. Click Save from the toolbar.
