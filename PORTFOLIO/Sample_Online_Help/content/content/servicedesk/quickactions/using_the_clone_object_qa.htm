Using the Clone Object Quick Action

You can use the Clone Object quick action to copy and overwrite the specified fields from a business object record. You can set up the quick action to only overwrite certain fields on a business object record, as shown in the following example of the Clone Object quick action in the Incident workspace:

1. Within the Configuration Console, open the Quick Actions Center for the incident business object.
2. From the Create New Actions tab, choose Clone Object from the list of options.
3. Enter information into the fields.

Clone Object Quick Action

-fields-
Field Description
Set Action Name Enter a name for this quick action, such as Clone Quick Action.
Assign a Category Select a category from the drop-down menu, such as Form Actions.
Publish to Select the roles which need access to this action (administrator, Change Manager, for example).
4. From the Field Name drop-down list, choose Status, then from the Field Value drop-down list, select a value for the Status field or enter an expression that will prompt the user to select a status for the copied incident, such as $(Prompt("Please select a Status for the new Incident", "")).
5. Click Add to add a row.
6. From the Field Name drop-down list, choose Owner. This defaults the Owner field to the user who is cloning the incident. For the corresponding Field Value, enter $(CurrentLoginId()).
7. Click Add to add a row.
8. From the Field Name drop-down list, choose Team. This defaults the Team field to the user's team. For the corresponding Field Value, enter $(CurrentUserTeam()).
If you do not want values from fields such as Cost and Total TimeSpent copied, select the fields from the Field Value section and enter 0 for the corresponding field values.
9. Click Add to add a row.
10. From the Field Name drop-down list, choose LastModBy to update the modified date. For the corresponding Field Value, enter $(CurrentDateTime()).
11. Click Save .
