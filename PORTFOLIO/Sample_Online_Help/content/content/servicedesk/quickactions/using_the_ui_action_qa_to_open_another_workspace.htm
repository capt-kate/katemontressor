Using the UI Action Quick Action to Open Another Workspace

In the following example, you can use the UI Action quick action to open a change form from the Problem workspace.

1. Open the Quick Actions Center for the problem business object.
2. On the left side, select the Create New Actions tab, then click UI Action from the list of options. The UI Action Configuration page appears.
3. Enter information into the fields.
-fields-
Field Description
Set Action Name Enter a name for this quick action.
Assign a Category Select a category from the drop-down menu.
Publish to (under the Action Settings tab) Select which roles need access to this template (administrator and Problem Manager, for example).
Scope Enter the command: CurrentTab:ObjectWorkspace:ObjectView.
Command id NewChildObject.
Confirmation null.
Command Data

Enter the following:

{

ObjectType:"Problem#"

, ChildObjectType:"Change#"

, NewObjectType:"Change#"

}

In another example, from the same Problem workspace, if you wanted a new announcement window to open, for CommandData, you would enter:

{

ObjectType:"Problem#"

, ChildObjectType:"Announcement#"

, NewObjectType:"Announcement#"

}
4. Click Save to save the quick action.
