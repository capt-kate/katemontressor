Canceling a Release

A new release record is in a state of draft. You can leave it in this state while you are gathering information to enter. When you are ready to submit the release, update its status to requested. If you decide to cancel the release, you can do so before you update the release status to requested, unless you are the Release Manager.

If you decide to cancel the release when it is in active state, you must also cancel its associated milestones otherwise the workflow associated with each milestone goes into effect. See Cancel a Release in Active State.

You can only cancel a release when its in draft state, unless you are the Release Manager.
Cancel a Release in Draft State
1. Log in as Change Manager or Release Manager and open the Release workspace.
2. Open the release record that you want to cancel.
3. Choose Action Menu > Cancel Release.
4. In the Enter Notes for Cancellation field, enter your reason for canceling the release and click OK. The status of the release is set to canceled.
5. Click Save. The Release Manager and the release owner are notified of the cancellation.
Cancel a Release in Active State
1. Log in as a Release Manager and open the Release workspace.
2. Open the release record that you want to cancel.
3. Choose the Milestones tab of the release record.
4. Select the first milestone record in the sequence and click Go to.
5. Choose the Workflow Instance tab and select the workflow instance record.
6. From the Action Menu, select Abort. The workflow instance status updates to aborted.
7. On the Milestone form, set the Status to Cancelled.
8. Click Save. The Release Manager and the release owner are notified of the cancellation.
