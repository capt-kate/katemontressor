Release Stakeholders

Users in the application can be identified as stakeholders for a Release. You can select a user and link that user to the Release record. The user is notified as being the stakeholder for the Release. Stakeholders might have a stake in the outcome of the Release and work with the Release Approval Board to make sure that approved processes are followed in implementing and deploying a Release.

The stakeholder function is to make sure that the authorized process is being followed for the Release, that a backout plan is in place, that the rollout into the live environment will have the least possible risk and negative impact, and so on.

There is no approval process by default for Release stakeholders. They are merely notified when their employee record is linked to a Release record.

Add a Stakeholder to a Release
1. Log in as Release Manager and open the Release to which you want to identify stakeholders.
2. Choose the Stakeholder tab.
3. Click Link. A list of employee names appears.

Search for the employee and click Search.

For example, you can search for employees whose First Name Begins With the letters sh.

A list of employees matching your search criteria appears.

4. Select the employee you want to link as a stakeholder and click Select. The employee is linked to the Release record as a stakeholder.
5. Repeat to add more employees.
6. Click Save.

The employees you selected as stakeholders are notified through email of the Release.

Set up PRBs

Each PRB (Problem Review Board) is a container that has individual members. Each PRB is associated to a problem of specific priority. A notification is sent to each member of the PRB when:

  • The root cause of a major problem is identified.
  • A major problem is resolved.
  • A priority 1 problem is logged.
  • A service is linked to the problem.

View or Modify Predefined PRBs

1. Log in as the Problem Manager.
2. Click Problem Review Board from the toolbar.

Problem Review Board

A list of the PRB records appears.

3. Select the record that you want to view or modify, and open it.
4. Update any of the fields.
5. Link or remove the PRB members for the record that you are viewing. See Add Members to a PRB.
6. Click Save.

Add a New PRB

1. Log in as the Problem Manager.
2. Click Problem Review Board from the toolbar.

Problem Review Board

A list of the PRB records appears.

3. Click New Contact Group.
4. Enter a unique name for Group Name.
5. From the Group Type menu, select Problem Review Board.
6. From the Problem Priority menu, select the type of problem that should be reviewed by this PRB.
Make sure that another PRB is not linked to the same problem priority. If there is already a PRB record for the problem priority that you are selecting, edit the existing record instead of creating a new one.
7. Select a Status, either active or inactive.
8. Click Save.

Add Members to a PRB

1. Add a new PRB or view an existing PRB record.
2. On the Member tab, click Link.
3. Search for an employee and click Search.
4. Select the employee name from the list and click Select.
5. Add other team members to the PRB by following steps 3 and 4.
6. To remove a PRB member, select the record and click Unlink.
7. Click Save.
View Stakeholders for a Specific Release
1. Log in as the Release Manager and open a release record.
2. Choose the Stakeholder tab. A list of the users that have been identified as stakeholders is displayed.
