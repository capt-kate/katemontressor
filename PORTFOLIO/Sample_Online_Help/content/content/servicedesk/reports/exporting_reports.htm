Exporting a Report

You can export a report to a text file and use it within other applications, such as Microsoft Excel or Microsoft Word.

 

Restrictions for Exporting Data Reports

You can create reports that automatically export data using the Export Data Quick Action. HEAT comes with templates that you can modify to export your data into a file (such as XML or CSV) on your local computer.

  • Only use the Export Data quick action to export table (grid) reports.
  • Use single value text parameters to pass the current context data to report.
  • Do not export reports with hidden parameters.
  • You can only export up to 10,000 records.
Export Report Data

Follow these steps to export a report:

1. Open a report.
2. Select a DateTime parameter from the Start Date and End Date menus.
3. If you selected multiple reports, in the View Report window, click the navigation arrow to view the reports.
4. From the View Report window, do any of the following to export the report to another application:
  • From the Select a format menu select a format in which to view the report.
  • Use the arrow keys, if necessary, to page through the report data.
5. Click Export. You can view, save, or print the output file.

 

Create a Report

Create a report as described in Creating Reports and Templates using the following parameters:

-parameters-
   
Report Template Choose the template you modified from the drop-down list (such as ExportIncident).
Publish Report To Choose one or more roles to access the report with the quick action.
Can Run from Export Data Quick Action Check this option.
Parameter Information

The parameters are automatically populated from the report template. For example, for the ExportIncident template, the TimeZoneOffset and CurrentBrowserCulture fields appear.
No Schedule Check this option. This report will be assigned to a quick action which will run on an ad-hoc basis.
Report Format Choose the format you want for your report from the drop-down list.

Click Save from the toolbar.

Create a Quick Action
1. After your report template has been customized, log back into HEAT as administrator (or other role with administrator privileges), and open the workspace to which you want to use the quick action (such as incident, task, employee, CI, or service request).
2. From the Action Menu, choose Edit Actions. The Quick Action Configuration window appears.
3. From the Create New Actions tab, choose Export Data. A blank form appears.
4. Enter information as explained in Export Data Quick Action using the following parameters:
-parameters-
   
Report Name Choose the report you created from the drop-down list.
Can Run for Multiple Records Check this option.
Connection

Choose the connection you created from the drop-down list.

  • Cloud users can only use an FTP connection.
  • Premise users can only use a file share connection.
Schedule Leave this field empty.
Report Parameters The parameters from the report are automatically generated. In the Source Value field, you can use a field, enter an expression, or type a specific value.
5. Click Save.

You are now ready to run the quick action.

Run the Quick Action
1. Open the workspace for which you created the quick action and refresh the screen.
2. From the Action Menu, choose No Category > <your_report_name>.

The report runs.

Monitor Progress

To monitor the progress of the report, open the Run Program History workspace.

Run Program History Workspace

The report is automatically logged on this page. If it is a long report, you might need to click Refresh to see the updated status of the report.
