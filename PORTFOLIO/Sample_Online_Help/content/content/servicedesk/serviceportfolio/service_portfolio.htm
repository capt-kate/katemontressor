Limited Edition FeatureService Portfolio

The Service Portfolio represents a complete list of the services managed by the service provider. It contains present contractual commitments, new service development, and retired services.

The Service Portfolio workspace makes it easier for you to introduce a new service into your environment. You can follow service development through the stages of Defined, Analyzed, Approved, and Chartered.

Access Service Portfolio
1. Log into the Service Desk Console as Service Portfolio Manager or administrator, and open the Service Portfolio workspace. The list of defined services appears (see Creating a Service).
2. Double-click an item from the list to view or modify its information.
Create a Service project plan

You can create a detailed project plan for managing all milestones and tasks needed to drive a service development or transition process through various stages.

1. From the Service Portfolio workspace, click New Service from the toolbar. A blank form appears.
2. Enter your information into the fields.
-fields-
Field Description
Description Enter a unique name for this project.
Stage

Choose from the drop-down list.

  • Define - The initial stage. Set the project definitions.
  • Analyze - The project is in the analyze and evaluate stage.
  • Approve - The project is in the approval stage.
  • Charter - The project is in the charter stage.

The graph automatically corresponds to the project stage.
Service Requirements Type a detailed explanation of the business requirements for this project.
3. Select the Service Introduction Project tab. The list of introduction projects appear, if any.
4. Click Save from the toolbar.
5. Select the Service Introduction Project tab. A list of project plans appear, if any.
6. From the New Service Introduction Project drop-down list, choose New Service Introduction. The Edit Release window appears.

As part of the service introduction project plan, you should create a Milestone called Create Service Design Package and link the related documents such as SLA, SLR, and service packages as attachments.

As part of the service transition plan, you could include a service improvement plan and service quality plan and attach it to the milestone.

7. Enter your information into the fields. See Create a Release Request for more information.
8. Click Save from the toolbar.

Once the project is created, you can define, manage and track the process of the Service Introduction from the release project.

Link to supporting services
1. Within a Service Portfolio record, select the Support Services tab. A list of linked services appear, if any.
2. Click Link. The list of available services appear.
3. Choose a service from the list, then click Select.
4. Click Save from the toolbar.

The number of linked services automatically appears in the tab.

Attach business case records or web pages
1. Within a Service Portfolio record, select the Business Case Records tab. A list of attached documents or URLs appear, if any.
2. To attach a document or file, click New Attachment. Navigate to the desired file, then click OK.
3. To link to a web page, click New URL. Enter the web address (ie. http://www.frontrange.com), then click Save.
4. Click Save from the toolbar.

The number of linked documents or web pages automatically appears in the tab.
