Service Request Escalations

Various events are monitored by an escalation clock.

SLAs associated with a service override any system escalations when service level agreements are associated with a request item. In the absence of service level agreements, default escalations are run. See Creating an Escalation.

If an event does not occur within a prescribed period of time, an escalation action is executed. The exact timing and actions of an escalation depend on how your system is configured.

Escalations typically provide notification in a dashboard or service request form. It is up to the Service Desk Analyst or Manager to take appropriate action based on such notifications.

If SLAs are configured for a service, then the escalations that are configured within the SLA overrides the default system escalations. If another service is selected, the previous escalation clock is stopped and the records of the previous escalation schedule are marked obsolete. A new escalation schedule goes into effect.

You can view response and delivery escalations directly on the Resolution Details tab of the service request form.

The response target uses the ServiceReq response escalation schedule, and the delivery target uses the ServiceReq delivery escalation schedule. If SLAs are configured for a request offering, then the escalations that are configured within the SLA override the default system escalations.

 

ServiceReq Response

When a service request is submitted by a requester through the Self Service module, its initial status is set to Submitted. The status of the service request is changed to another state when it is worked on. The amount of time that the service request is in the Submitted state is the Service Request response time. If service request response time exceeds a default duration, a notification displays on the Service Request dashboard. If the service request is created and saved by a Service Desk Analyst in the HEAT client, the service request response escalation is considered complete.

To respond to a breach:

1. Within the Service Desk Console, open the Service Request workspace. The list of service requests appears.
2. Double-click the service request that was identified in the escalation notification. The service request record appears.
3. In the Owner field, select a name from the drop-down list.
4. Click Save.
  • The status of the service request is updates to Active.
  • The service request is considered responded to and the escalation ends.
  • A journal entry records these activities in Activity History.
  • The end user is notified (if automatic notification is enabled).
ServiceReq Delivery

The amount of time that a service request is in the Submitted, Active, Approved, or Waiting for 3rd party state prior to being fulfilled or closed is the Service Request Delivery time. When the service request fulfillment or closed time exceeds the time defined for a type of request offering to be resolved, its status is displayed in the resolution target area of the Service Request form. Service request delivery can provide up to four increasingly urgent levels of escalation—1st, 2nd, 3rd, and breach.

To end a ServiceReq delivery escalation, update the status of the service request to fulfilled or closed.

Task Response

If you or your team is assigned a task, you must respond to it within a prescribed amount of time. If you do not, an escalation notification displays as specified by your system configuration.

When you are notified of a task response escalation:

1. Open the service request to which the task is linked.
2. Click the Task tab.
3. Select the escalated task (only tasks with no status or a status of assigned or logged will be escalated).
4. Click one of the following buttons to begin processing the task and end the escalation:
  • Accept
  • Complete
5. Reassign (the task reverts to a Logged state)
  • Continue
  • Cancel
  • Waiting
6. Click Save.
Task Resolution

To be considered resolved, a task must have a status of Completed or Cancelled. If a task is not resolved in a prescribed amount of time, an escalation notification displays as specified by your system configuration.

When you are notified of a task resolution escalation:

1. Open the service request to which the task is linked.
2. Open the escalated task (only tasks with no status or a status of assigned or logged will be escalated).
3. Click one of the following buttons to resolve the task:
  • Complete
  • Cancel
  • Waiting
4. Click Save.
ServiceReq Waiting

If the service request is in approved, submitted, or waiting for customer status, the service request waiting escalation runs. To respond to this escalation, close or cancel the service request.

ServiceReq Closing

If there is no further communication from the requester upon receiving notification about a fulfilled service request, the service request is automatically closed after three business day (as configured by default). The closure time can be customized to meet your organization's business rules.

View Service Request Escalations
  • View escalations from the Resolution Details tab of a service request record. View details on each escalation from the Escalation Watch tab.

If SLAs are configured for a service, then the escalations that are configured within the SLA overrides the default system escalations.

If the delivery target bar or the escalation records on the Escalation Watch tab do not appear, click the Refresh button on the Service Request toolbar.
