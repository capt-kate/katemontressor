Adding the Social Board to a Role

To expose Social Service Management based on a role, first make the Social Board workspace available for selection in the Service Desk Configuration tab, and then use the Service Desk Configuration tab to grant Social Board workspace access to a role:

1. Within the Configuration Console, open the User Interfaces workspace to view available roles. Select the role to see the available tabs and workspaces for that role.

User Interfaces Workspace

2. If the Social Board workspace is available, you can optionally change its display order by dragging it up or down the workspace list. To manage the visibility of the workspace, click Advanced Options and adjust the properties as needed.
3. Click Add User Interface.

Add User Interface

4. Select the Social Board workspace.

Social Board Workspace

5. Name the Social Board tab and define other properties as necessary.

Social Board Properties

6. Save your changes.

The Social Board workspace is now visible for the Service Desk Configuration tab to enable access to the Social Board workspace for one or more roles.
