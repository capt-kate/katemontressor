Creating a Task

While working on a parent object such as an incident, a change, a problem, or a release milestone, you can create tasks to complete work on the parent object. These tasks can then be assigned to different owners.

1. Within the Service Desk Console, open a parent object. For example, an incident record.
2. From the Task tab, click New Task. The New Task window appears. The parent object name and the parent ID automatically appear on the task record and the task is in logged status.
3. Enter information into the fields.
-fields-
Field Description
Summary Enter a brief summary title for the task.
Details Describe the scope of the task.
Status This field is automatically generated.
Priority The priority is automatically entered, or you can choose from the drop-down list.
Service Select a service organization from the drop-down list.
Team Select a team from the drop-down list.
Manager This field is automatically generated based on the team field.
OLA

This field appears according to the chosen team. This field is automatically generated.

When fulfilling an incident or request of a service with an OLA, the SDAService Desk Analyst will see the defined task catalog. If there is no OLA, the task catalog does not appear.

If a SDAService Desk Analyst is completing a form,

Task Catalog This field appears according to the chosen team. Choose a task catalog from the drop-down list.
Owner Select an owner from the drop-down list.
Email This field is automatically generated bases on the Owner field.
Phone This field is automatically generated based on the Owner field.
Actual Effort Enter the amount of time in minutes the task has taken.
Cost This field is automatically generated based on the actual effort and cost per unit.
4. Click Attach file to attach a document to the record.
5. To add a comment, click Create. See Notes and Comments.
6. Click Save.

At this time:

  • The task record appears under the Task tab of the business object.
  • The task owner (if specified) is notified of the assignment.
  • The team manager (if the owner is not specified) can assign the task to a team member.
  • The task status is updated to assigned if an owner is selected, otherwise it remains in logged status.
