Task Escalations

Various events are monitored by an escalation clock.

If you are not using Service Level Management, defined escalations are set to run.

If an event does not occur within a prescribed period of time, an escalation action is executed. The exact timing and actions of an escalation depend on how your system is configured.

Escalations typically provide notification to the Task Owner and Team. It is up to the Owner and Team Manager to take appropriate action based on such notifications.

View Task Escalations

Tasks are configured with default escalation schedules. Use caution when modifying Task schedules and workflows as this might cause unexpected results.

When a Task is created and saved, the escalation clock starts.

You can view escalations for Tasks:

  • From the escalation schedule on a progress bar on the Task record.
  • From the Escalation Watch tab of the Task record.
View Escalation Details
1. Within a Task record, select the Details tab.

The task Response Target and Completion Target bars appear at the bottom of the page.

The Response Target uses the Task.Assignment Response escalation schedule, and the Resolution Target uses the Task.Assignment Resolution escalation schedule.

2. View details on any of the escalations under the Escalation Watch tab.
3. Double-click an item from the list to view escalation details.
Default Task Escalation Schedules
  • The default Task Response escalation schedule is one day.
  • The default Task Resolution escalation schedules are as follows:
