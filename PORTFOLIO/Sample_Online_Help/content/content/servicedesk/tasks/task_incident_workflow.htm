Use > Incident > Task-Incident Workflow

* * * New or Improved for 2013.2 * * *  Task-Incident Workflow

Incidents containing tasks need to have agreement before completion. In other words, an incident should not be able to be closed if it contains open tasks. Also, once all tasks within an incident are closed, then the incident should be automatically resolved, or a new task automatically created. However, an incident can be closed regardless of the tasks status, automatically closing any open tasks.

Incidents cannot be updated while in logged status.

Enable the Task-Incident Workflow

This feature is not enabled by default; users must manually enable this option to use it. To enable this feature, you must set the Task.Assignment and incident Validation On Save rules.

1. Within the Configuration Console Main Menu, choose Business Objects, then open the Incident business object page.
2. Choose the Business Rules tab. The list of rules appears.
3. At the bottom of this list is the Validation On Save Rules section. The only rule listed is shown crossed out (disabled).

Validation On Save Rules

The full expression is:

$(((Status != "Closed") && (Status != "Resolved")) || IsTrueForAllChildren("Incident#", RecId, "Task#Assignment.Rev3", "$(Status == 'Completed' || Status == 'Cancelled' || Status == 'Rejected')"))

Click Enable next to the rule. The rule is no longer crossed out.

4. Click Save on the toolbar to commit your settings.
5. Return to the Home page and open the Task.Assignment business object, then choose the Business Rules tab. The list of rules appears.
6. At the bottom of this list is the Validation On Save Rules section. The only rule listed is shown crossed out (disabled).

Validation On Save Rules Section

The full expression is:

$(IfAllSiblings("Incident#.Rev3", "$(Status != 'Closed' && Status != 'Resolved')", "$(Status == 'Completed' || Status == 'Cancelled' || Status == 'Rejected')", "$(PromptAndExecuteAction('$(Parent(Incident#.Rev3))', 'Task belongs to an Incident with no other unfinished Tasks. Select [Yes] to set Incident resolved, select [No] to create a new Task, or select [Cancel] to cancel the Task save operation.', 'Resolve Incident', 'New TaskAssignment'))"))

Click Enable next to the rule. The rule is no longer crossed out.

7. Click Save on the toolbar to commit your settings.
Quick Action Configuration

The TaskWorkflow quick action category contains one default action: New TaskAssignment.

Task Workflow

This quick action checks the incident-task workflow validation rule when the record is saved.
