* * * New or Improved for 2014.2 * * *  Universal Worklist

The universal worklist feature enables you to simultaneously view worklists from multiple tenants. A worklist is a list of items (usually issues, changes, or problems) and is typically the result of a saved search.

To use this feature correctly, the following conditions apply:

  • The databases must all be in the same landscape (such as domain or sub-domain).
  • The databases must all contain the same saved search that you want to use.
  • Each tenant user can be a slave or a master tenant. The master tenant can see all slave tenants, but slave tenants can only see the master tenant.
  • Each slave tenant must be able to provide an API key and URL. (See DSM Integration for an example of what this looks like.) The master tenant does not need to provide this.

Do not use hyperlinks in grids used with the universal worklist. Hyperlinks cause the user to double click to view record details. However, the command always goes to the master tenant, which then fails.
Create a Universal Worklist

The universal worklist is created as a dashboard part and can be added to a new or existing dashboard.

To enable this feature, you must:

  1. Obtain the URL and APIThe Application Program Interface (API) is a set of routines, protocols, and tools for building the software application. key of each slave tenant you want to access.
  2. Define the Physical Structure: Create a materialized view of a business object.
  3. Define the Source Data: Obtained from the slave tenants.
  4. Define a Grid: Prepare to show the data.
  5. Create a Saved Search: Needed to create a dashboard part.
  6. Define the Dashboard Part: View the final data.
Define the Physical Structure
  • Within the Configuration Console, create a Materialized View business object. See Create a Materialized View Business Object for information on how to do this.
Define the Source Data

The actual data is created and updated only in the tenants, not the Materialized View business object. The materialized view shows the synchronized data from the tenants.

  • Define the source data from the View Parts tab of the Materialized View business object. See View Parts for information on how to do this.
Define a Grid

To use the universal worklist in a dashboard, you must be able to use it in a saved search. To view saved search results, you must create a grid.

  • You can create a grid from scratch (see Using Grids) or from the View Details tab, click Auto Generate UI. Edit the grid to include the columns you need.
Create a Saved Search
1. From the Configuration Console, click Build > Saved Searches, then click Show All Objects.
2. From the materialized object that you created, click Add. The Saved Search window appears.
3. Create a saved search as described in Using Saved Searches
To show all records in your search results, add a date parameter that is earlier than your known data. For example, where Created On is Greater Than January 1, 1970.
Define the Dashboard Part

The dashboard part is where you view the final data.

1. Create a new dashboard part. See Dashboards for information on creating a dashboard.
2. Expand the Special Parts accordion, then click Add MSP list . The Create New MSP Dashboard Part window appears.
3. Enter information into the fields.
-fields-
Field Description
Name Enter a unique name for your dashboard part, such as Change Universal List.
Description Optional. Describe what the part is for.
Object Select a business object from the drop-down list.
Saved Search

Select a saved search from the drop-down list. This field is populated by the saved searches in the change object. Ensure this saved search is common among all tenants you want to access.
Grid Select the grid that you want from the drop-down list. For example, the Change grid is a generic list.
Tenant URL Enter the URL of a tenant you want to access. This tenant must be in the same domain that you are currently in.
Key Enter the API key of the tenant that you want to access.
4. Click Test Connections. If the connection status is successful, the Connected column shows yes.
5. To add a tenant click Add, then enter the URL and API key, then click Test Connections.
6. Click Save to save the dashboard part, then drag it onto the dashboard.
7. Click Save from the toolbar.

A list of records from all tenants appears. Double-click a record from the list to view it in the corresponding tenant.

Data Security

Data is restricted by Role. To provide a restricted view, create a segregation rule. See Apply Advanced Security Features for more information.

1. From the Configuration Console, click Configure > User and Permissions > Roles and Permissions. The list of roles appears. See About Roles for more information on this feature.
2. Choose the role to which you want to grant access, then select the Object Permissions tab.
3. Find the object that you created, then select the options that you want.

Add a segregation rule to the master role.

Use the Universal Worklist
1. Log into the Service Desk Console as the role you have given permissions to see the universal worklist, then open the dashboard you created. (It is under the My Dashboards list.)
2. Click Refresh to synchronize the data and allow it to appear. The first time you do this might take a minute.

The current user can view all tenants, but can only see data from the current or assigned tenants.

The color of the title bar indicates how recent the data is. A red title bar means the data is more than 10 minutes old. A blue title bar means the data has been synchronized within the last 10 minutes.
