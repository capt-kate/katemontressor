Sync Remote Control Session Quick Action

This quick action syncs the data between users during a Remote Control session. Available only in the Incident workspace.

Use this quick action to sync the remote session when events outside the application occur. For example, remote control automatically populates a start and end date for a session. When a session is forcibly closed (say, by closing the Remote Control window from the SDAService Desk Analyst page or by killing the Remote Control client), the end date is not recorded. The connection in these cases was closed outside the system, and therefore the system has no way of knowing what happened or that it should update the end date. To fix this, run the Sync Remote Control Session quick action, and the session data is then synchronized.

This quick action contains no editable fields.

See Remote Control.
