Extended Task Workflow Block

Extend Task Workflow Block

Exit Ports

  • Define additional exit ports under the Exits Configuration tab.
accepted The task status is set to accepted.
completed The task status is set to completed.
canceled The task status is set to canceled.
timed out

The workflow is terminated as incomplete because an event exceeded the defined length of time.

Properties

Title Enter a unique name for the object.
Task Type Choose from the drop-down list.
Copy from predefined task
Service Optional. Choose from the drop-down list.
Team Choose a team from the drop-down list.
Owner Choose an owner from the drop-down list.
Relationship to Task Choose a relationship from the drop-down list.
Notification Quick Action Choose from the drop-down list.
Timeout Field To set a timeout based on a field, select this option, then choose a field from the drop-down list.
Timeout Duration To set a timeout based on duration, select this option, then enter the number of days, hours, and minutes you want.
Hours of Operation Name To set a timeout constrained by hours of operation, select this option, then choose the name of your hours of operation from the list. See Setting Up Hours of Operation.
Hours of Operation Field To set a timeout constrained by an hours of operation field, select this option then choose a field from the list.
Task Type Enter a literal value or choose an object from the drop-down list. The available fields and target object fields appear based on the chosen object.

Actions

  • Use the Move Up and Move Down arrows to rearrange table items and the order in which they are processed. The order of processing can affect the creation or update of the business object, as specific values are determined to be correct or not.
  • Check Overwrite to overwrite default values with this value.
  • Click Save to commit your settings.
