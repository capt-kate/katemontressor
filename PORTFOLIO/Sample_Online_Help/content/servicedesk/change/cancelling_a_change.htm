Canceling a Change Request

A new change request has a status of logged. If you want to cancel a change request, you must do so before you update the status to requested.

Follow this procedure to cancel a change request that you created, while it is still in the logged state.

Unless you are a Change Manager, you can only cancel a change request when it is in the logged state. Change Managers can change the status of a change request at any time.
Canceling a Change Request that is in the Logged State
1. Log in to the Service Desk Console as a Change Manager and open the Change workspace. Open the change request that you want to modify.
2. In the Status field, choose Canceled from the drop-down list.
3. Click Save from the toolbar. The Cancellation Reason window appears.
4. Enter the reason for canceling the request and click OK. The status of the change request is set to canceled.

The Change Manager and the change requester are notified via email of the cancellation.

Canceling a Change Request that is in a State Other Than Logged

Follow the steps in Canceling a Change Request that is in the Logged State and then continue here:

1. Click the Workflow Instance tab and select the workflow instance record, if any.
2. Click Abort from the Workflow Instance toolbar. The workflow instance status is set to aborted.
