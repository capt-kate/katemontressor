Setting up a Change Window and Lockout

The Configuration Manager can create a lockout period and a change window for a CI. The lockout period appears on the change scheduler in red, while the change window appears in green.

A change lockout period indicates that changes should not be made during this time. They can be overridden by the Change Manager, if necessary.

A change window indicates to the Change Manager that it is a preferred time to implement a change for the CI.

Set up a Change Window

By setting up a change window for a CI, you are letting Change Management know of the preferred time to implement changes that affect the CI. For example, you can set up a change window during weekends or non-work hours.

1. While logged into HEAT as Configuration Manager or administrator, open the Configuration Item workspace. The list of objects appears.
2. Double-click the record you want from the list. The record details appear.
3. Select the Change Window tab, then click Link. A list of schedules appears.
4. Choose the schedule you want, then click Select. The schedule appears in the list.
5. Click Save from the toolbar.
Create a New Schedule
1. Within the Change Window tab, click New Change Window. A blank change window form appears.
2. Enter information into the fields.
--fields--
Field Description
Name Enter a unique, descriptive name for the schedule.
Description Optional. Describe how the schedule is to be used.
Start Time / End Time
All Day Event Check this option if you do not want to specify specific times.
Start Date / End Date Select the start and end dates (according to your local time) of the change lockout.
No End Date Check this option if you do not want to specify an end date.
Daily To run the change lockout daily within the above specified dates, enter the number of days. For example, recurs every 2 days.
Weekly To run the change lockout weekly within the above specified dates, enter the number of weeks. For example, recurs every 1 week on Wednesday.
Monthly To run the change lockout monthly within the above specified dates, enter the number of months. For example, recurs every 3 months on the second Monday.
Yearly To run the change lockout annually within the above specified dates, enter the number of years. For example, recurs every 2 years in January on the first Monday.
3. Click Save.

If this CI links to a change, the Change Manager can view the change window and know the preferred time to implement any changes that involve this CI.

Link CIs to a Change Window

You can associate a single CI or multiple CIs to a change window. If a change is created using any of the associated CIs, the change window is shown in the Change Calendar tab of the change record.

1. While logged into HEAT as administrator or Configuration Manager, open the Change Window workspace. The list of objects schedules appears.
2. Double-click the object you want from the list. The record details appear.
3. Select the CI tab, then click Link . A list of records appears.
4. Select the record you want, then click Select.
5. Click Save from the toolbar.

The CIs are now associated with the change window. If a new change is created for any of the CIs that are associated with this change window, the change window appears in the Change Schedule tab of the change record. The CI records also show the change window associated with the CIs.

Set up a Change Lockout

By setting up a change lockout for a CI, you are letting Change Management know that changes that affect the CI should not be made during the lockout window. For example, you can set up a change lockout during business hours on weekdays.

1. While logged into HEAT as administrator or Configuration Manager, open the Configuration Item workspace. The list of objects appears.
2. Double-click the object you want from the list. The record details appear.
3. Select the Change Lockout tab, then click Link . A list of schedules appears.
4. Select a schedule and click Select. The schedule appears in the list.
5. Click Save from the toolbar.
Link CIs to a Change Lockout

You can also associate a CI or bulk CIs to a change lockout from the Change Lockout workspace.

1. While logged into HEAT as administrator or Configuration Manager, open the Change Lockout workspace. The list of objects schedules appears.
2. Double-click the object you want from the list. The record details appear.
3. Select the CI tab, then click Link . A list of records appears.
4. Select the record you want, then click Select.
5. Click Save from the toolbar.

The CI(s) are now associated with the change lockout. If a new change is created for any of the CIs that are associated with this change lockout, the change lockout appears in the Change Schedule tab of the change record. The CI records also show the change lockout associated with the CIs.
