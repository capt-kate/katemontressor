CI Costing

By default, Cost Management is integrated with Configuration Items (CIs) and CI services. In HEAT, this feature is available only to the Configuration Manager and administrators.

The Configuration Manager can add both direct item costs and indirect (or supporting) costs to a configuration item. In doing so, the total cost of providing a Service associated with a CI can be tracked, thus enabling budgeting and auditing of IT spending.

The Cost History tab records each time cost history is run. You can configure a workflow that determines how frequently cost history is run.

Adding or updating a schedule entry can be done by administrators only. After adding or updating a costing schedule entry, the schedule entry must be associated with the costing workflow and the existing Run Web Service block must be updated to access the correct URL. Both of these tasks are done in the Configuration Console. If these two tasks are not done, your cost history will not run.

Item costs are costs that are incurred directly with the service; while indirect costs are the costs that are associated with all the CIs or CI services that are linked to the specific CI (shown in the CI Map Tree tab) in a downstream relationship.

CI components or CIs that are downstream to the current CI are shown in the Indirect Costs (Current Month) table.

Direct costs or the cost of the specific CI are reflected in the Direct Costs table, while the roll up costs of all the associated CI items appear in the Indirect Costs (Current Month) table.

From the Direct Costs table, you can also change the cost allocation of the current CI to its upstream CI(s).

The Total field in the Summary (Current Month) table shows the total of all direct costs and the total of all indirect costs for the current month.

You cannot directly add cost to the CIs shown in the Indirect Costs (Current Month) table. To add cost information to any of the associated CIs, search to open the CI record to add the cost information.

Add Direct Costs to a CI
1. Log into the Service Desk Console as a Configuration Manager or administrator and open the CI or Configuration Item workspace.
2. Open the CI record for which you want to add costing.
3. Select the Cost Items tab, then click Add Cost Item.
4. Enter information into the fields.
-fields-
Field Description
Name Enter a name for this cost item, for example, Maintenance Cost.
Cost

Enter the dollar amount of this cost item.

You can also enter an expression, enclosed in $() symbols. The expression is applied in the context of the main business object and fields in the main business object can be referenced. For example, you can enter an expression of $(iif(Status=='Retired', 0, 5000)), which means that if the Status of the CI is Retired, the cost is $0, otherwise the cost is $5000.
Recurrence

Select if this is a one-time cost or a recurring cost. The options for recurring cost are Monthly and Yearly.

For a one-time cost, the end date is not calculated. The Summary (Current Month) table shows the date referenced in the Start Date and the cost is incurred once, for that month only. The cost is not carried over the next month.

For a monthly recurring cost, the start date is the date referenced in the Start Date field for the current month and then calculated for each month on the scheduled frequency until the month of the end date. The Summary (Current Month) table shows the date referenced in the Start Date and the cost is incurred each month until the End Date.

For a yearly cost, the Start Date is the date referenced and the expense is incurred once each year. The following year's expense is incurred on the same month and the same expense recurs once a year until the End Date.

The item cost is updated at the specified recurrence and added to the cost history, which updates at the configured frequency.

5. Repeat to add more cost items.

Add Cost to an Associated Downstream CI

All CIs in a downstream relationship to the current CI service are shown in the Indirect Costs (Current Month) table.

1. Log into the Service Desk Console as a Configuration Manager or administrator and open the CI or Configuration Item workspace.
2. Open a CI record, then in a new workspace, open the associated CI record for which you want to add costing.

For example, Web Hosting Service, Data Server, ERP SQL DB, NA-SAN-STORAGE, and PL-DATA-SERVER are some of the 54 CIs and Services that are downstream to the current CI Service—the HRIS service, as shown in the following screen:

Notice that the Contribution column for the HRIS row shows a roll up of all the costs, taking into account the cost allocation.

The ERP SQL DB database does not have any associated costs.

3. In the associated CI, select the Cost Items tab, then click Add Cost Item. Enter cost information into the fields.
4. In the original CI, click Update Cost History, then click Refresh Indirect Costs. The costs appear as specified.
5. To change the allocation, enter a different percentage amount in the Cost Allocation column..
Change the Rollup Distribution of the Current Service

The HRIS system is downstream to the Employee Administration system.

You can allocate what percentage of HRIS will be distributed to the Employee Administration System,

1. Within a CI record, click Edit Rollup Distribution.
2. Click the Cost Distribution field and enter the percentage, for example 20%, that should be allocated to Employee Administration, then click OK.

If you open the Employee Administration record, notice that the allocation from HRIS is 20% of its total cost items of $700, which is $140.

View Cost History

The frequency with which cost history is run is configurable. By default, the cost history is scheduled to update each month on the first of the month. However, you can create a new schedule entry record and associate the schedule entry with the CI Cost Service workflow.

You can also manually run cost history. Direct costs and all supporting costs are recalculated each time cost history is run. Each time cost history is run (whether manually or at the scheduled frequency), the direct and supporting costs are generated only for those CIs currently open or for the CI in which cost history is manually run. If the CI contributes to the rollup cost of another CI, then that will not update.

Only Configuration Managers and administrators can add, edit, and view costing information.

Manually Run Cost History

To manually run cost history, click Update Cost History, then click OK to dismiss the message.

Manually running cost history generates the cost history for that CI only. The current month's history for the CI is deleted and rewritten.

View Cost History

Click the Cost History tab, then click Refresh History.

The list of records show the direct and indirect costs of the service.

Modify the Cost History Scheduling Frequency
Adding or updating a schedule entry can be done by the administrator only. After adding or updating a costing schedule entry, the schedule entry must be associated with the costing workflow and the existing Run Web Service block must be updated from the localhost URL to the appropriate URL. Both these tasks are done in the Configuration Console.

By default, the CI Cost Service schedule entry is based on a monthly recurring schedule on the first of each month, which means that the workflow runs on the first of each month and updates the cost history information.

After you define a new schedule entry or modify an existing one, the administrator must associate the new schedule with the CI Cost Service workflow.

Associate a Schedule Entry with the Cost Workflow

1. Within the Configuration Console, open the Schedule Entry business object. The Object Details appear.
2. Select the Workflows tab. The list of workflows appear.
3. Click CI Cost Service. The CI Cost Service workflow versions appear.
4. Click Edit next to the last version. The workflow designer appears.
5. Click Configuration from the top of the designer. The Workflow Configuration window appears.
6. For Use Schedule Definition, choose CI Cost Service.
7. Click Save, then click Save again.

CI Cost Service

The new workflow definition is not published until you click the Publish button and confirm to publish the new definition.

8. Click Publish to make the new workflow definition available to users.

In the Configuration Console, the administrator must also update the WSDL URL from localhost to the correct tenant name, as shown in the following screen:

Modify the WDSL URL

1. In the Configuration Console, go to the Schedule Entry business object.
2. Click the Workflows link from the toolbar.
3. Click the CI Cost Service link.
4. Click the Edit button.
5. Double-click the Run Web Service block.
6. Click OK to dismiss the error message Cannot load web service definition.
7. In the WDSL URL field, double-click to select the localhost/HEAT URL and replace it with the correct tenant name.
8. Save and close the window.
9. Save the workflow.

The new workflow definition is not published until you click Publish and confirm to publish the new definition.

10. Click Publish to make the new workflow definition available to users.
11. Click Yes in the Version Publish Confirmation window to publish the new workflow.
