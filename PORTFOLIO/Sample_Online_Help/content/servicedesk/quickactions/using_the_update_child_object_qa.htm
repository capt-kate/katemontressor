Using the Update Child Object QA

You can use the Update Object Quick Action to update certain fields on a child business object. For example, you can update the Status field on a Task record when a Change record is set to Implemented.

1. Open the Quick Actions Center for the Change Business Object.
2. From the Create New Actions tab, select Update Object from the list of options.
3. Enter your information into the fields.
-fields-
Field Description
Name Enter a name for this Quick Action.
Category Select a category from the drop down menu. For example, Form Actions.
Publish to Select the roles which need access to this action. For example, Change Manager.
Hide when

Enter a command to hide this Quick Action under certain conditions.

For example, if you do not want the Quick Action to be enabled when the Change record is in Logged or Closed state, enter the following command in the Hide when field: $(iif( (((Status == "Closed") || (Status == "Resolved")) || (Status == "Logged")), true, false))
4. From the Action settings tab, next to Apply action to, click Change, then select ChangeContainsTask:Task#Assignment.

By default, the action is applied to the Change (selected object). However, in this instance because you are creating this Quick Action for Change Tasks, click Child Object.

Click OK to close the Apply to window.

5. Enter information into the Field Name and Field Value sections to customize the update object action:
  • For Field Name select Status.
  • For the corresponding Field Value, click the folder next to the field to open the Edit Expression window, then type the following:
    $(iif((nvl([top.]Status, "") == "Implemented"), "Completed", Status))  
6. Click Save to save the expression.
7. Click Save to save the Quick Action.
