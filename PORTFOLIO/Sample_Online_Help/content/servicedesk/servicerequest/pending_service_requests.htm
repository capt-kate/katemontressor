Pending Service Requests

You might want to create a service request, but do not have all the required information. If the required information is not entered, the service request cannot be saved. However, you can save a draft, or Pending, version of the service request.

Create a Pending Service Request
1. Log into the Service Desk Console as a service owner and open the Service Catalog workspace. The list of services appear.
2. Click an item to create a new service request. The service request wizard appears.
3. Enter as much information as you can.
4. At the bottom of the page, click Save for later. The service request is now saved in pending mode.
5. At the bottom of the page, click Cancel to close the window and return to the Service Catalog workspace.

Service Catalog Workspace

Pending service requests do not initiate a workflow nor execute a notification.

View Pending Requests
1. Log into the Service Desk Console as a service owner and open the Service Catalog workspace. Any pending requests appear in a pane on the right side and a Pending Requests link appears in the workspace search area.

Pending Requests

If the pane does not appear, click Pending Requests from the Search area. The number of requests in your queue is indicated.

Pending Requests Queue

2. Click an item from the list to view and edit as needed. Missing information is indicated in the wizard.

Pending Request Item

Once request information has been submitted, it disappears from the pending list.

Delete a Pending Request
  • Click the Delete button next to a pending request to permanently remove it.

Delete a Pending Request
