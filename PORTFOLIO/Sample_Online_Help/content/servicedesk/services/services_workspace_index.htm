Services Workspace Index

The following table lists the workspaces related to services, agreements, request offerings, and service requests. The main workspaces used are highlighted.

Workspace Name Description Used for
Operational Level Agreements List of OLAs for all services. To define, go to Services > Service Agreements > New Operational Level Agreement.
Request Offerings List of services offered. Define service forms and workflows.
Services List of all services. Configure services. Create and link teams, vendors, categories, and SLPs.
Service Catalog List of service offered by category. Create requests for service. Request offerings appear here to users.
Service Level Agreements List of SLAs for all services. To define, go to Services > Service Agreements > New Underpinning Service Level Agreement.
Service Level Target List of SLTs for all agreements. To define, go to Services > Service Level Packages, then Go To a specific package.
Service Level Package List of SLPs for all services. To define, go to Services > Service Level Packages.
When an organizational unit subscribes to an SLP, an SLA is created. The SLA inherits the targets from the SLP.
Service Request Available to SDM or SDA. End users create requests for service.
SLA Subscription Create agreement packages. Link agreements and subscribe to services. Link agreements and subscribe to the service agreement tab. These are the service-based SLAs that will document all customers who are using this service. The related customer-based SLAs are linked to the employee profile record.
Underpinning Contract List of UCs for all services. To define, go to Services > Service Agreements > New Underpinning Contract.
Vendor Administrator only. List of Vendors used in all services. Can create vendors to use for external contacts and tasks.
