* * * New or Improved for 2014.1 * * *  Using a Cascading Validation List

This example illustrates how to trigger values from additional fields or a combo box (cascading validation list) by creating a service that uses a value based on the field OwnerTeam.

Before configuring this option, ensure you have a standard user team called Desktop Support that it is populated with active employee names.

Standard User Team - Desktop Support

1. Log into the Service Desk Console as an administrator, and open the Request Offering workspace. The list of request offerings appears.
2. Enter "overnight" in the Search Service Offerings text box. The Overnight Standby Request offering appears.

Request Offering - Overnight Standby Request

3. Click Overnight Standby Request. The Define Request Offering page appears.

Sample Request Offering - Define Request Offering Page

4. Click the Design Request Form tab. The service request form appears.

Sample Request Offering - Design Request Form Page

5. Click Edit next to Service. The Configuration Editor appears. (See Configuring the List Control for more information.)

Sample Request Offering - Item Configuration Editor for Service

6. Expand the Pick List: Validation List area, and in Validation List to Use, choose Service Name List from the drop-down list.
7. In the Additional fields section, add the Service business object and choose the OwnerTeam field.

Sample Request Offering - Service Business Object, OwnerTeam Field

8. Click OK.
9. Click Edit next to Service Support Team Member. The Configuration Editor appears again.

Sample Request Offering - Item Configuration Editor for Service Support Team Member

Ensure the Cascading Validation Definition shows Standard User Team object and the OwnerTeam field.

10. Click OK.

When the end-user selects Overnight Standby Request: Desktop Service from the Service Catalog, the service field OwnerTeam automatically populates the Service Support Team Member list from the Employee Team: Desktop Support list. The OwnerTeam must be previously configured to include team members. (See Using Teams.)
