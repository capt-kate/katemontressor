Activating Field Security Permissions

Photos of users in the Social Board are stored using several fields in the Profile.Employee business object. To successfully upload a photo, you must provide edit permissions to the fields relative to a given role.

For example, to activate the necessary field permissions for Self Service users, do the following:

1. From the Configuration Console, select Roles and Permissions. The list of roles appears.
2. Select Self Service. The Role Details page appears.
3. Select the Object Permissions tab, then from the Select Page drop-down menu, navigate to the employee business object.
4. Click the arrow next to Employee to expand the list of permissions.

Field-Level Permissions

5. Check Edit permissions (check the second column) for the following fields in the employee business object:
  • EmployeeInformation
  • EmployeePhoto
  • EmployeePhotoName
  • EmployeePhotoRevision
  • LastModBy
  • LastModDateTime

Edit Permissions

6. Click Save to persist the field level permission changes.

Social Service management functionality is now activated in existing customer databases.

The following roles have edit permissions activated by default for the Profile.Employee business object:

  • Admin
  • Business Unit Manager
  • Change Manager
  • Executive
  • Knowledge Manager
  • Problem Manager
  • Release Manager
  • Self Service
  • Service Desk Analyst
  • Service Desk Manager
  • Service Owner

Field permissions must be specifically activated for any other roles not mentioned in this list.
