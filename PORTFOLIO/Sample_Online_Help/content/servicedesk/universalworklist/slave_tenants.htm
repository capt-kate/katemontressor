Slave Tenants

This workspace defines slave tenants to use with the universal worklistThe ability to view a list of items from multiple user workslists on one workspace.. See Universal Worklist.

Administrators can view slave tenantPart of a multi-tenant data configuration, used in the universal worklist. information, unless you create a segregation business rule. See Using Business Rules for information on how to do this.

Creating a Slave Tenant
1. Log into the Service Desk Console as an administrator, then open the Slave Tenants workspace. The list of slave tenants appears, if any.
2. From the toolbar, click New FRS_SlaveTenants. A blank Slave Tenants form appears.
3. Enter information into the fields.
--fields--
Field Description
API Key The slave tenant client authentication key. For example, 06B0916645894B8184644C00B700B001.
Tenant Url The slave tenant URL. For example, http://company.com/computer_0047
4. Click Save from the toolbar.
