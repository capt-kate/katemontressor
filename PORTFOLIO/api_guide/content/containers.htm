Containers

Containers are a consistent, buildable and deployable artifact. They make standardized development and deployment easy, and can provide operational efficiencies. Shipped uses Docker, a popular container format, for builds and deployments. Some understanding of how to use Docker for development might be helpful as you integrate with Shipped. Once your Docker container works, you can automatically build and deploy your code.

Learn more about Docker and containers at Docker.com

Containerize your app

Building the right container to run your application in depends on the specifics of your application.

  1. Choose a base image
  2. Install dependencies
  3. Add build commands
  4. Tell Docker how to run the app

An example Ruby on Rails Dockerfile

FROM ruby:2.2.0
RUN apt-get update -qq && apt-get install -y build-essential libpq-dev
RUN mkdir /myapp
WORKDIR /myapp
ADD Gemfile /myapp/Gemfile
ADD Gemfile.lock /myapp/Gemfile.lock
RUN bundle install
ADD . /myapp
CMD rails s
